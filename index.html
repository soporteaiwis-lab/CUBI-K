Conceptos Clave

Ideas fundamentales discutidas en la sesión que todo participante debe dominar

🤖
Gema (Asistente)
Un asistente de IA preconfigurado dentro de plataformas como Gemini. Funciona bien para tareas simples y consultas directas, pero tiene limitaciones cuando la asistencia se vuelve compleja o requiere múltiples pasos automatizados.
⚙️
Agente de IA
Va más allá de un asistente. Requiere su propia aplicación con backend y automatización. Es más escalable que una gema y puede ejecutar tareas complejas de forma autónoma. Se programa con herramientas como N8N, Zapier o Make.
🔗
Contextualización
El paso más difícil al usar IA: proporcionar el contexto adecuado para obtener respuestas precisas. Sin buen contexto, la IA genera información genérica o incorrecta. Es la habilidad #1 para dominar como usuario de IA.
📓
Notebook LM
Herramienta de Google que actúa como repositorio de conocimiento. Permite crear cuadernos con fuentes de datos, hacer consultas inteligentes, generar resúmenes y hasta videos. Se usará intensivamente en el curso.
🔄
Tokens y Limitaciones
Las versiones gratuitas de herramientas como Claude o Gemini permiten usar los modelos, pero limitan la cantidad de tokens (unidades de texto) o tiempo de uso, causando bloqueos temporales. Las versiones Pro eliminan estas restricciones.
🏢
"Gemflix" — Ecosistema Interno
Concepto meta de la Fase 1: crear una especie de "Netflix de asistentes de IA" dentro de CUBI-K, con una galería de herramientas IA especializadas para tareas repetitivas del equipo (análisis, documentación, pipelines, etc.).

🔍 Comparativa: Gema (Asistente) vs Agente de IA

🟡 Gema / Asistente

  • → Funciona dentro de la plataforma (Gemini, ChatGPT)
  • → Responde consultas y genera texto
  • → Configuración limitada a prompts
  • → No tiene backend propio
  • → Ideal para tareas simples y puntuales
  • → Nivel 1 en la curva de aprendizaje
VS

🟢 Agente de IA

  • → Aplicación independiente con backend
  • → Ejecuta acciones y automatizaciones
  • → Integra múltiples fuentes y APIs
  • → Escalable y personalizable
  • → Gestiona tareas complejas autónomamente
  • → Nivel 2+ en la curva de aprendizaje

Anotaciones del Mentor

Tips y Notas Destacadas

Consejos prácticos y reflexiones importantes compartidos durante la sesión

TIP 01
La contextualización es la habilidad #1
El mayor obstáculo al usar IA no es la herramienta, sino la capacidad de darle el contexto correcto. Sin contexto, las respuestas serán genéricas. Con buen contexto, la IA se vuelve un experto personalizado.
TIP 02
Usa tu cuenta personal de Gmail
Las cuentas corporativas de BCI no tienen habilitado Notebook LM ni acceso completo a herramientas. Usar cuentas personales garantiza acceso libre a todas las funcionalidades sin restricciones del banco.
TIP 03
Las versiones gratuitas sí funcionan
Claude, Gemini y otras herramientas gratuitas usan los mismos modelos que las de pago. La limitación está en los tokens y tiempo de uso. Una cuenta Pro de Claude puede compartirse entre compañeros para reducir costos.
TIP 04
De consultar a ejecutar: el cambio de mentalidad
La mayoría del equipo usa IA para "hacer preguntas". El salto está en pasar de preguntar a que la IA ejecute tareas directas: generar scripts, procesar datos, automatizar flujos completos.
TIP 05
Notebook LM como base de conocimiento
Crear un cuaderno en Notebook LM desde el inicio del curso permite acumular conocimiento progresivamente. Se puede consultar, generar resúmenes, crear mapas mentales y hasta producir videos con el contenido.
TIP 06
Google Sheets + App Script = Apps gratis
El portal del curso usa Google Sheets como base de datos con extensiones de App Script. Esto demuestra que se pueden crear aplicaciones completas con herramientas gratuitas, un concepto que se profundizará en Fase 2.
